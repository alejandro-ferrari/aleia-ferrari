# Plan a project (spec-first)

1. Clarify the GOAL (not just the task) and its success criteria — ask 1–2 questions only if
   genuinely unclear.
2. Produce a concise plan: problem, success criteria (binary/checkable), scope (in / out),
   steps, open questions, risks.
3. Save to `outputs/plan-<slug>.md` (generated).
4. Self-review (report pass/fail with evidence): every section present; success criteria are
   binary; assumptions named.

Keep it a plan to act from, not a manual. Reply in the user's language.
